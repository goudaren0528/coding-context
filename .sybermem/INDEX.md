# SyberMem Index — ctx (AI Workspace Runtime)

> Local-first, no cloud. `.sybermem/` is the canonical project data directory.

---

## Key Conclusions

- The product is a **CLI runtime layer** (`ctx`) that wraps Claude Code / OpenCode with persistent project memory and cross-project developer preferences.
- Architecture follows a **three-tier memory model**: Workspace State (volatile), Project Brain (slowly-evolving), Developer Memory (cross-project, in `~/.ctx/`).
- Storage is **local SQLite** via `sql.js` (WASM-based, no native compilation needed). Project DB at `<repo>/.ctx/db.sqlite`, user DB at `~/.ctx/db.sqlite`.
- **State-centric** design: session compression merges findings into workspace state instead of storing independent summaries ("where am I?" > "what did I say?").
- **L1-L4 layered prompt injection** to avoid token bloat: L1 terminal header, L2 project state, L3 project brain, L4 developer preferences.
- **No personality profiling** — developer memory records coding preferences (libraries, patterns, workflows) only.
- **Tech stack**: TypeScript + Node.js 22+, Commander CLI, Anthropic SDK for LLM extraction, `child_process.spawn` for PTY passthrough.
- **Phase 1 (complete)**: 28 source files, 5 CLI commands (`claude`, `opencode`, `resume`, `memory`, `update`), project auto-init + attach, session lifecycle, LLM compressor.
- **Phase 2 (complete)**: memory_candidates table with approve/reject/forget workflow and auto-approval (≥3 projects), cross-project pattern detection in `ctx update`, workspace_state_versions table, enhanced preference filtering by language/projectType/tool, decision decay with importance_score halving after 30 days.
- **Phase 3 (complete)**: privacy redaction (14 regex patterns for API keys, tokens, JWTs), `ctx doctor privacy`, `ctx state edit`, file exclusion patterns, session event sanitization before storage and LLM.
- **Phase 4 (complete)**: `ctx export` / `ctx import` for sanitized developer memory portability, `ctx restore` for environment recovery (branch checkout, file listing), `ctx ask` for natural language semantic retrieval over project memory using Claude API.
- Design doc at `docs/design.md` (v0.3.0, locked).
